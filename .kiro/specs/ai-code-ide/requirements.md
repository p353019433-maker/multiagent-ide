# AI Code IDE - 需求文档

## 项目概述

构建一个带 AI Agent 能力的桌面代码编辑器，支持多供应商 LLM 接入和自定义 endpoint，让用户可以通过 AI 自主完成代码编写、修改和调试任务。

## 技术栈

- **框架**: Electron + React + TypeScript
- **编辑器内核**: Monaco Editor
- **构建工具**: Vite
- **样式**: Tailwind CSS

---

## 功能需求

### FR-1: 文件管理

**用户故事**: 作为开发者，我需要浏览、打开、编辑和保存项目文件。

**验收标准**:
- [ ] 可以打开本地文件夹作为工作区
- [ ] 左侧显示文件树，支持展开/折叠目录
- [ ] 支持多标签页编辑
- [ ] 文件修改后显示未保存标记
- [ ] 支持 Ctrl/Cmd+S 保存文件
- [ ] 支持创建、删除、重命名文件和文件夹

### FR-2: 代码编辑器

**用户故事**: 作为开发者，我需要一个专业的代码编辑体验。

**验收标准**:
- [ ] 基于 Monaco Editor 提供代码编辑
- [ ] 支持语法高亮（主流语言）
- [ ] 支持代码折叠
- [ ] 支持多光标编辑
- [ ] 支持查找和替换
- [ ] 支持主题切换（至少深色/浅色）

### FR-3: AI 供应商管理

**用户故事**: 作为用户，我需要配置不同的 AI 供应商和模型，以便灵活选择。

**验收标准**:
- [ ] 支持预设供应商：OpenAI、Anthropic、DeepSeek、Google Gemini
- [ ] 支持自定义 endpoint（任何 OpenAI 兼容 API）
- [ ] 每个供应商可配置：名称、Base URL、API Key、默认模型
- [ ] API Key 使用 Electron safeStorage 加密存储
- [ ] 可以在多个供应商之间快速切换
- [ ] 支持配置多个模型（每个供应商下可选不同模型）
- [ ] 连接测试功能：验证 API Key 和 endpoint 是否可用

### FR-4: AI 聊天面板

**用户故事**: 作为开发者，我需要与 AI 对话来讨论代码问题和获取帮助。

**验收标准**:
- [ ] 右侧/底部可切换的聊天面板
- [ ] 支持 Markdown 渲染（含代码块高亮）
- [ ] 流式输出（逐字显示 AI 回复）
- [ ] 可以将当前文件或选中代码作为上下文发送
- [ ] 聊天历史持久化
- [ ] 支持多轮对话
- [ ] 可以新建/切换对话

### FR-5: Agent 模式

**用户故事**: 作为开发者，我希望 AI 能自主执行任务——读取文件、修改代码、运行命令——而不仅仅是给建议。

**验收标准**:
- [ ] AI 可以调用工具（Tool Use / Function Calling）
- [ ] 内置工具集：
  - `read_file`: 读取指定文件内容
  - `write_file`: 创建或覆盖文件
  - `edit_file`: 对文件进行局部编辑
  - `list_directory`: 列出目录内容
  - `run_command`: 在集成终端执行命令
  - `search_files`: 在项目中搜索文本
- [ ] Agent 执行过程可视化（显示调用了哪些工具、参数、结果）
- [ ] 用户可以中断 Agent 执行
- [ ] 文件修改前显示 diff 预览，用户可接受/拒绝
- [ ] Agent 执行有超时保护

### FR-6: 集成终端

**用户故事**: 作为开发者，我需要在 IDE 内运行终端命令。

**验收标准**:
- [ ] 底部面板集成终端（基于 xterm.js）
- [ ] 支持多终端标签
- [ ] 终端工作目录跟随当前工作区
- [ ] Agent 的 run_command 工具在此终端执行并显示输出

### FR-7: 设置界面

**用户故事**: 作为用户，我需要一个直观的界面来管理 IDE 和 AI 配置。

**验收标准**:
- [ ] 设置页面/对话框
- [ ] AI 供应商配置区域（增删改供应商）
- [ ] 编辑器设置（字体大小、主题、Tab 宽度等）
- [ ] 快捷键查看

## 非功能需求

### NFR-1: 性能
- 启动时间 < 3 秒
- 打开大文件（>1MB）不卡顿
- AI 流式响应延迟 < 500ms（网络正常时）

### NFR-2: 安全
- API Key 加密存储，不以明文出现在配置文件中
- 不将用户代码发送到除用户配置的 AI 服务外的任何第三方

### NFR-3: 跨平台
- 支持 macOS、Windows、Linux

### NFR-4: 可扩展性
- 工具系统设计为可插拔，方便后续添加新工具
- 供应商适配器模式，方便接入新的 AI 服务

优先实现以下核心功能：
1. ✅ 文件树 + Monaco 编辑器（FR-1, FR-2）
2. ✅ AI 供应商配置（FR-3）
3. ✅ AI 聊天面板 + 流式输出（FR-4）
4. ✅ Agent 模式基础版（FR-5）
5. ⏳ 集成终端（FR-6，可简化）
6. ⏳ 设置界面（FR-7，可简化）
